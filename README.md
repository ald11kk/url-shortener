# Project 1: URL Shortener API

A minimal REST API that shortens long URLs, built with FastAPI and Pydantic. This is the first project in a backend engineering learning roadmap.

---

## What It Does

- Accept a long URL and return a short code
- Redirect visitors from the short code to the original URL
- Return a `404` if the short code doesn't exist
- (Stretch) Track and return visit counts per short URL

## API Endpoints

### `POST /shorten`

Accepts a long URL and returns a short code.

**Request body:**
```json
{
  "url": "https://www.example.com/some/very/long/path"
}
```

**Response:**
```json
{
  "short_code": "abc123",
  "short_url": "http://localhost:8000/abc123"
}
```

---

### `GET /{short_code}`

Redirects to the original URL with a `301` status code.

**Success:** `301 Redirect → original URL`

**Failure:** `404 Not Found` if the code does not exist

---

### `GET /stats/{short_code}` *(stretch goal)*

Returns visit statistics for a short URL.

**Response:**
```json
{
  "short_code": "abc123",
  "original_url": "https://www.example.com/some/very/long/path",
  "visit_count": 42
}
```

## Getting Started

### Prerequisites

- Python 3.12+
- [uv](https://github.com/astral-sh/uv) installed

### Installation

```bash
# Clone the repo
git clone https://github.com/your-username/url-shortener.git
cd url-shortener

# Install dependencies
uv sync
```

### Running the Server

```bash
uv run uvicorn app.main:app --reload
```

The API will be available at `http://localhost:8000`.

Interactive docs (auto-generated by FastAPI) at `http://localhost:8000/docs`.

uv run pytest tests/ -v

## Implementation Notes

- Short codes are random 6-character alphanumeric strings
- Storage is in-memory (a Python dictionary) — data is lost on restart
- URL validation uses Pydantic's `AnyHttpUrl` type — invalid URLs are rejected automatically
- In-memory storage will be replaced with PostgreSQL in Project 2

---

## Learning Goals

This project covers the core backend request lifecycle:

1. Client sends an HTTP request
2. FastAPI routes it to the correct handler
3. Pydantic validates the input — bad data is rejected before it reaches your logic
4. Business logic runs and reads/writes data
5. FastAPI serializes the response and sends it back

These fundamentals apply to every backend system, regardless of scale.
